# Lessons —— 纠正与教训累积

> 每次被用户纠正/发现自己漏的，立即在此追加：**发生了什么 → 哪里错了 → 以后的规则**。
> session 开始先读这里。这是复利系统：每条都降低未来犯同样错的概率。

---

## L1 · 别凭记忆给用户安需求

- **发生**：收尾时我声称阈值通知"在你的需求里"，还提议去实现。
- **错在**：用户其实一直说要**删掉**它，从没要求过。我凭（被压缩过的长会话的）记忆臆断了需求。
- **规则**：写"你要过 X / 这是需求"前——尤其在交接文档里或提议（重）建某功能时——先 grep 用户原话/代码找依据。找不到依据就说"我可能记错了"并确认，**不要断言**。

## L2 · 重写脚本后，同步所有文档里的行号引用

- **发生**：把续命从 `claude -p` 改成 OAuth refresh，刷新器从 ~57 行涨到 ~90 行。
- **错在**：`ARCHITECTURE.md` / `HANDOVER.md` 里约 15 处 `xxx.sh:NN` 行号引用全失效，还有一个文档引用的函数名 `token()` 在新代码里根本不存在。
- **规则**：改了脚本**行数**，立刻 grep 一遍所有 `:NN` 行号引用与函数名，逐一对照真实文件修正。交接前用审计核一遍。

## L3 · 对外文案只说真话，不夸大

- **发生**：早期文案有"完全零消耗""（隐含）纯本地"的倾向。
- **错在**：默认模式要联网调 Anthropic 用量端点，不是纯本地；早期 `claude -p` 续命也并非零额度。
- **规则**：宣称前先核实成立性。**不写"纯本地/完全离线"**（仅桥接模式纯本地）；"零额度"只在确实零额度时说。首屏主打真实稀缺差异（**只读用量不读对话**），而非人人都有的"免费/开源"。

## L4 · 动 keychain 凭证前先备份，改后验证结构

- **发生**：实现零额度续命要写回 `Claude Code-credentials`，而该 blob 里还有用户的 `mcpOAuth`（vercel/flomo）token。
- **规则**：动 keychain 写入逻辑前先 `... -w > 备份`；写回只改 `claudeAiOauth` 三字段、deep-copy 保留其余；改后读回验证 `mcpOAuth` 等结构完整、token 已更新、CC 不被登出。refresh_token 会轮换，故必须写回。

## L5 · 调试别打印 token；功能未实现别宣称

- **发生**：调试 OAuth refresh 时把新 token print 到了输出（事后靠再轮换一次作废）；落地页曾宣称"原生 macOS 通知"，但代码里根本没有通知逻辑。
- **规则**：token 只读进变量，要打印只打长度/尾 4 位。**对外只宣称代码真实具备的功能**；撤掉任何无代码支撑的声明。
